In a computer's memory hierarchy, the **main memory** (typically DRAM) acts as the central hub. Both **cache** and **auxiliary memory** serve to optimize this system but reside on opposite ends of the performance and capacity spectrum.

### 1. Characteristics Comparison

| **Feature**    | **Cache Memory (vs. Main Memory)**                           | **Auxiliary Memory (vs. Main Memory)**                       |
| -------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Speed**      | **Faster**: Designed to match the processor's speed, reducing wait times. | **Slower**: Significantly slower access times (often by orders of magnitude). |
| **Capacity**   | **Smaller**: Limited by high cost and physical space on or near the CPU. | **Larger**: Provides massive storage for files and programs not currently in use. |
| **Cost**       | **Higher**: Much more expensive per bit of storage.          | **Lower**: The most cost-effective way to store large volumes of data. |
| **Volatility** | **Volatile**: Loses data when power is removed.              | **Non-volatile**: Retains data permanently without power.    |
| **Placement**  | Integrated into the CPU chip or placed very close to it.     | Connected via I/O interfaces (SATA, NVMe, USB).              |

### 2. Specific Examples of Memory Devices

#### **Cache Memory: SRAM (Static Random Access Memory)**

Cache is almost exclusively implemented using **SRAM**.

- **Why:** SRAM does not need to be "refreshed" like DRAM (main memory), allowing for near-instantaneous access. It uses a flip-flop circuit (typically 6 transistors per bit) to store data, which makes it very fast but physically bulky and expensive, explaining why cache sizes are relatively small (KB to MB).

#### **Auxiliary Memory: SSD (Solid State Drive) or HDD (Hard Disk Drive)**

Auxiliary memory is commonly represented by an **SSD** (using NAND Flash memory) or an **HDD** (using magnetic platters).

- **Why:** These devices are non-volatile and can store Terabytes (TB) of data at a fraction of the cost of RAM. While main memory holds the data the CPU is currently processing, the auxiliary memory acts as long-term storage for the operating system, applications, and user files.

# 2-3

To find the CPI and the speedup ratio, we need to calculate the total cycles per instruction by adding the base execution cycles to the stall cycles caused by cache misses.

### (2) Finding the CPI for Program B

The total CPI is calculated using the following formula:

$$\text{CPI}_{\text{total}} = \text{CPI}_{\text{base}} + \text{Memory Stalls per Instruction}$$

**1. Determine the Base CPI ($\text{CPI}_{\text{base}}$):**

The problem states that for Program A, which induced no memory stalls, the CPI was **1.50**. Since we are using the same computer for Program B, the base execution cycles (excluding stalls) remains the same:

$$\text{CPI}_{\text{base}} = 1.50$$

**2. Calculate Memory Stall Cycles for Program B:**

Memory stalls occur for both Instruction Cache misses and Data Cache misses.

- **Instruction Stall:** Every instruction requires a fetch.

  $$\text{Inst Stall} = \text{Miss Rate}_{\text{inst}} \times \text{Miss Penalty} = 0.01 \times 100 = 1.0 \text{ cycles/inst}$$

- **Data Stall:** Only load-store instructions (60% of total) trigger data cache accesses.

  $$\text{Data Stall} = \text{Percentage}_{\text{LS}} \times \text{Miss Rate}_{\text{data}} \times \text{Miss Penalty}$$

  $$\text{Data Stall} = 0.60 \times 0.04 \times 100 = 2.4 \text{ cycles/inst}$$

**3. Total CPI for Program B:**

$$\text{CPI}_B = 1.50 + 1.0 + 2.4 = \mathbf{4.90}$$

------

### (3) Finding the Speedup Ratio with Secondary Cache

When a secondary (L2) cache is added, a miss in the primary (L1) cache first checks the L2. A penalty is only 100 cycles if it misses both L1 and L2 (accessing main memory). If it misses L1 but hits L2, the penalty is only 10 cycles.

**1. Define the new Stall Components:**

The "rate of accessing main memory" given (0.5%) represents the **Global Miss Rate** ($MR_{\text{L1}} \times MR_{\text{L2}}$).

- **Instruction Stalls:**
  - L1 miss, L2 hit: $(\text{L1 Miss Rate} - \text{Global Miss Rate}) \times \text{L2 Penalty} = (0.01 - 0.005) \times 10 = 0.05$
  - L2 miss (Main Memory): $\text{Global Miss Rate} \times \text{Main Memory Penalty} = 0.005 \times 100 = 0.5$
  - $\text{Total Inst Stall} = 0.05 + 0.5 = 0.55 \text{ cycles/inst}$
- **Data Stalls (for 60% of instructions):**
  - The problem implies the memory access rate for data was also reduced to 0.5%.
  - L1 miss, L2 hit: $0.60 \times (0.04 - 0.005) \times 10 = 0.60 \times 0.035 \times 10 = 0.21$
  - L2 miss (Main Memory): $0.60 \times 0.005 \times 100 = 0.30$
  - $\text{Total Data Stall} = 0.21 + 0.30 = 0.51 \text{ cycles/inst}$

**2. New CPI with Secondary Cache ($\text{CPI}_{\text{new}}$):**

$$\text{CPI}_{\text{new}} = 1.50 + 0.55 + 0.51 = 2.56$$

**3. Calculate Speedup Ratio:**

The speedup is the ratio of the execution times (or CPIs, since instruction count and clock cycle time are constant):

$$\text{Speedup} = \frac{\text{CPI}_B}{\text{CPI}_{\text{new}}} = \frac{4.90}{2.56} \approx \mathbf{1.914}$$

The secondary cache achieved a speedup ratio of approximately **1.91**.
